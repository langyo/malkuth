# تسليم المستمع

> **الحالة: مُخطَّط.** تنشيط المقابس غير مُنفَّذ بعد. يصف التصميم أدناه
> السلوك المقصود.

## المشكلة

عند إعادة تشغيل عملية خادم، تكون هناك نافذة زمنية لا يستمع فيها أحد على
المنفذ — فتُسقَط الاتصالات الواردة. لتحديثات متداولة دون توقف، يجب على
العملية الجديدة أن ترث مقبس الاستماع من العملية القديمة.

## الحل: تنشيط المقابس

يُبقي systemd (أو مُطلِق مخصّص) واصف مقبس الاستماع مفتوحاً. عند إعادة تشغيل
العملية، ترث العملية الجديدة الواصف وتستطيع قبول الاتصالات فوراً — فالنواة
تُدخِرها في الطابور أثناء الفجوة.

واجهة API المُخطَّطة:

```rust
use malkuth::acquire_listener;

// يُفضّل تنشيط مقابس systemd (واصف موروث)، ويعود إلى ربط عادي.
let listener = acquire_listener("0.0.0.0:8080").await?;
```

فعِّل الميزة `socket-activation` (غير متاحة بعد):

```toml
malkuth = { features = ["socket-activation"] }
```

## كيف ستعمل

يضبط systemd متغيرَي بيئة:

| المتغير | المعنى |
| --- | --- |
| `LISTEN_PID` | مُعرّف العملية التي ينبغي أن ترث الواصفات (يجب أن يساوي مُعرّفنا) |
| `LISTEN_FDS` | عدد الواصفات المُمرَّرة (تبدأ من الواصف 3) |

سيقرأها Malkuth، ويتحقق أن `LISTEN_PID == our_pid`، ويأخذ ملكية الواصف 3
(`SD_LISTEN_FDS_START`)، ويضبطه على وضع عدم الحجب، ويغلّفه بـ
`tokio::net::TcpListener`.

إن كانت المتغيرات غائبة أو المُعرّف غير مطابق، يعود إلى
`TcpListener::bind(addr)`.

## مثال وحدة systemd

```ini
# /etc/systemd/system/myapp.socket
[Socket]
ListenStream=8080

[Install]
WantedBy=sockets.target
```

```ini
# /etc/systemd/system/myapp.service
[Service]
ExecStart=/usr/bin/myapp
```

بهذا الإعداد، لا يُسقِط `systemctl restart myapp` أي اتصال جارٍ: تُبقيه
النواة في طابور الاستماع أثناء بدء العملية الجديدة ووراثتها للواصف.
